# wexx

**在微信里,给你电脑上的 Codex 派活。**

发一条微信 → 电脑上的 Codex 开始干活 → 结果回到微信。

整理资料、处理文件、写文档、跑脚本,随时随地。

## 三步开始

**第 1 步:安装。** 把这句话发给 Codex:

> 帮我安装 wexx:https://github.com/focuxdot/WEXX

**第 2 步:绑定。** 在 Codex 里输入 `$wexx`,用微信扫码。

**第 3 步:派活。** 在微信里发出第一条消息,开始干活。

> 前提:macOS + 装好的 Codex + Node.js 18+。
> 不想让 Codex 装?终端里一行也可以:
> ```bash
> git clone https://github.com/focuxdot/WEXX.git ~/.codex/skills/wexx && cd ~/.codex/skills/wexx && npm install
> ```

## 微信里怎么用

发任何消息就是派活。看到"正在输入…"就是任务在跑。

只有这四条指令例外,它们不会发给 Codex:

| 发送 | 作用 |
|---|---|
| `停` 或 `/stop` | 立即中断当前任务 |
| `新任务` 或 `/new` | 开启全新会话 |
| `状态` 或 `/status` | 查看任务状态 |
| `帮助` 或 `/help` | 查看帮助 |

想解绑?对 Codex 说"解绑微信"。

## 安全

- 只响应**扫码绑定的那个微信号**。
- 任务**全速执行**:不逐条审批,可以读写文件和联网。首次绑定时会请你确认,只问一次。
- 所以,别派涉及你不信任内容的活——网页和文件里可能藏着恶意指令(提示注入)。
- 随时发 `停` 刹车。

## 限制

- 电脑休眠或关机时不干活(消息不丢,唤醒后继续)。
- 目前仅支持 macOS。
- 产出的文件保存在电脑上(`Documents/Codex/`),会告诉你路径;文件直接发回微信,在计划中。

## 它是怎么工作的

```
手机微信 ⇄ 腾讯官方 Bot 通道 ⇄ 你的电脑(wexx)⇄ 本机 Codex
```

- **没有第三方服务器**:消息直连你的电脑,本项目没有后端。
- **不碰你的 Codex 配置**:不读写 config.toml,不涉及任何 API key。
- **数据全在本地**:会话和文件只存在你自己的机器上。
- 微信里的对话同步出现在 Codex 桌面版的会话列表,回到电脑接着聊。
