# WeChat 连接器

把用户的微信接入这台电脑上的 Codex:扫码绑定一次,之后用户在手机微信里发消息
派活,回复推回微信。本 skill 只负责连接管理;绑定完成后的消息往来由常驻
daemon 处理,与本 skill 无关。

## 使用方式

一切操作都通过同一个幂等脚本(在本 skill 目录下执行):

```bash
node scripts/connect.mjs status
```

**每次被调用时,先运行上面的 status 命令**,然后根据输出行动:

### 情况 A:输出「未绑定微信」且「首次绑定,需先获得同意」

1. 不要直接执行 login。先向用户请求一次性授权,文案参考(可按语境微调,
   四个要点缺一不可,语气自然、不用术语):

   > 绑定前需要你确认一件事(只问这一次):
   > - 扫码连接之后,需要你首先发送消息,创建会话;
   > - 为了让任务不被反复打断,微信派的活会全速执行——不逐条审批,可以读写文件和联网;
   > - 随时有刹车:微信里发 /stop 或者 /停 立即中断,发 /help 查看指令;
   > - 想撤销随时说一声"解绑微信"即可。
   >
   > 同意的话我就出二维码。

2. 用户明确同意后,执行:

```bash
node scripts/connect.mjs login --yes
```

3. 终端会显示二维码(同时弹出 PNG 图片),提示用户用微信扫码。命令会阻塞
   等待扫码结果(最长 5 分钟),成功后自动启动后台服务并设置开机自启。
4. 把命令输出的结果转述给用户。若用户拒绝授权,不要执行 login,直接结束。

### 情况 A2:输出「未绑定微信」且「此前已同意,无需再次确认」

用户之前授权过,不要重复请求同意,直接发起扫码:

```bash
node scripts/connect.mjs login
```

### 情况 B:输出「已绑定微信」

status 命令已自动完成一切(包括 daemon 掉线时的静默重启)。把状态转述给
用户即可,不需要额外操作。

### 情况 C:用户想解绑、换微信号或停止服务

- 解绑并移除自启:`node scripts/connect.mjs unbind`
- 仅停止后台服务:`node scripts/connect.mjs stop`

执行前向用户确认意图。

## 约束

- 本 skill 不读写 Codex 的 config.toml,不涉及任何模型或 API key 配置。
- 除上述三个命令外,不要以其他方式操作 daemon 或状态文件。
- 首次绑定未获得用户对完全放行模式的明确同意前,绝不执行 `login --yes`;
  同意只需一次,之后的扫码(重试、重连、解绑后重绑)不再重复询问。
